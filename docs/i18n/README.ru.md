## Что делает

Выберите два временных окна — **базовое A** и **подозрительное B** — и deltascope сравнит средние значения метрик из локальных архивов истории, оценит каждое изменение по полярности метрики и построит трёхуровневый отчёт: **диагноз → доказательства → полные данные**.

## Возможности

- **Движок диагностических правил** — 16 встроенных межметричных правил (свап-спираль, насыщение диска, переполнение очереди accept, OOM, горячее ядро, SYN-давление, обнаружение перезагрузки…). Каждое срабатывание даёт вывод простым языком, доказательства и следующие команды. Никаких синтетических баллов здоровья.
- **146 метрик, 5 категорий** — PSI, потери softnet, горячие точки по ядрам (автосворачивание), распределение состояний TCP, direct reclaim, LVM/MD. У шумных счётчиков — собственные пороги.
- **Отчёт с полными данными** — стабильные строки остаются видимыми, но приглушены; порядок строк стабилен; насыщенность фона ∝ |Δ|; появление ⊕ / исчезновение ⊖ помечены отдельно; якоря Top-5.
- **Всё — конфигурация** — каталог, правила и пороги в экспортируемом JSON с валидацией при загрузке. В `profiles/` — full/core.
- **Headless-режим** — `deltascope compare` печатает тот же отчёт текстом (ANSI) или JSON, код выхода 2 при регрессии: готов для cron.
- **Создан для изолированных сетей** — один статический бинарник, встроенные UI и графики, локальная аутентификация, без CDN, без телеметрии, без исходящего трафика.

## Быстрый старт

Готовые бинарники в [`dist/`](../../dist/): `linux-amd64` (ядро ≥ 3.2), `linux-arm64`, `linux-amd64-el6` (ядро 2.6.32).

Сборка из исходников (один раз, с интернетом):

```bash
make vendor && make test && make build
```

Развёртывание (референс Rocky Linux 9, возможно полностью офлайн):

```bash
RETENTION_DAYS=7 LISTEN_ADDR=0.0.0.0:8080 \
DSCOPE_ADMIN_USER=admin DSCOPE_ADMIN_PASS='...' ./deploy.sh
```

## Использование

```bash
deltascope serve   -listen :8080 -archive DIR -data DIR [-catalog F] [-rules F]
deltascope user    add|del|list <name>
deltascope catalog export > catalog.json
deltascope rules   export > rules.json
deltascope compare -a-start 2026-07-09T14:00 -a-end 2026-07-09T15:00 \
                   -b-start 2026-07-10T14:00 -b-end 2026-07-10T15:00 \
                   [-format text|json] [-all] [-no-color]
```

## Семантика сравнения

- Счётчики усредняются **как скорости** по окну (семантика pmdiff)
- Δ% = (B − A) / |A| × 100; оценивается только при `|Δ| ≥ порога` (глобально 15 %, для шумных метрик — свои)
- Полярность: `worse_up` / `better_up` / `neutral`
- A=0 → B≠0 отображается как ∞; метрики, отсутствующие с обеих сторон, молча пропускаются
- Появление ⊕ / исчезновение ⊖ — самостоятельные события

## Безопасность

PBKDF2-HMAC-SHA256 (600k итераций) · HMAC-подписанные stateless-сессии · ограничение попыток входа по IP · белый список имён метрик + exec-массив (никогда через shell) · строгая CSP без inline · усиленный systemd-юнит · учётные данные не покидают хост.

## Примечания

Окна трактуются в локальном часовом поясе сервера · окно до 32 дней · шаг тренда адаптивный · графики в комплекте (Apache-2.0) · в первый день архивы должны накопиться.
