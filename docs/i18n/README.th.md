## ทำอะไร

เลือกช่วงเวลาสองช่วง — **ฐาน A** และ **ช่วงต้องสงสัย B** — แล้ว deltascope จะเปรียบเทียบค่าเฉลี่ยของแต่ละเมตริกจากอาร์ไคฟ์ประวัติในเครื่อง ตัดสินทุกการเปลี่ยนแปลงตามขั้วของเมตริก และสร้างรายงานสามชั้น: **การวินิจฉัย → หลักฐาน → ข้อมูลทั้งหมด**

## คุณสมบัติ

- **เอนจินกฎวินิจฉัย** — กฎข้ามเมตริกในตัว 16 ข้อ (สไปรัลของ swap, ดิสก์อิ่มตัว, คิว accept ล้น, OOM, คอร์เดียวร้อนจัด, แรงกดดัน SYN, ตรวจจับการรีบูต…) ทุกครั้งที่ตรงเงื่อนไขจะให้ข้อสรุปภาษาคน หลักฐาน และคำสั่งถัดไป ไม่มีคะแนนสุขภาพสังเคราะห์เด็ดขาด
- **เมตริกในตัว 146 รายการ · 5 หมวด** — รวม PSI, softnet drops, จุดร้อนรายคอร์ (พับอัตโนมัติ), การกระจายสถานะ TCP, direct reclaim, LVM/MD ตัวนับที่แกว่งมากมีเกณฑ์เฉพาะ
- **รายงานข้อมูลครบ** — แถวที่ไม่เปลี่ยนยังมองเห็นแต่จางลง ลำดับแถวคงที่ ความเข้มพื้นหลัง ∝ |Δ| เกิดใหม่ ⊕ / หายไป ⊖ แยกชัด พร้อมจุดยึด Top-5
- **ทุกอย่างคือไฟล์คอนฟิก** — แคตตาล็อก กฎ และเกณฑ์เป็น JSON ส่งออกได้ ตรวจสอบตอนโหลด `profiles/` มีชุด full/core
- **โหมด headless** — `deltascope compare` พิมพ์รายงานเดียวกันเป็นข้อความ (ANSI) หรือ JSON ออกโค้ด 2 เมื่อพบการถดถอย: ใช้กับ cron ได้ทันที
- **ออกแบบเพื่อเครือข่ายปิด** — ไบนารีสแตติกตัวเดียว UI และกราฟฝังในตัว ยืนยันตัวตนในเครื่อง ไม่มี CDN ไม่มีเทเลเมทรี ไม่มีทราฟฟิกขาออก

## เริ่มต้นอย่างเร็ว

ไบนารีสำเร็จรูปอยู่ใน [`dist/`](../../dist/): `linux-amd64` (เคอร์เนล ≥ 3.2), `linux-arm64`, `linux-amd64-el6` (เคอร์เนล 2.6.32)

คอมไพล์จากซอร์ส (ครั้งเดียว บนเครื่องที่ต่อเน็ต):

```bash
make vendor && make test && make build
```

ติดตั้ง (อ้างอิง Rocky Linux 9 ออฟไลน์สมบูรณ์ได้):

```bash
RETENTION_DAYS=7 LISTEN_ADDR=0.0.0.0:8080 \
DSCOPE_ADMIN_USER=admin DSCOPE_ADMIN_PASS='...' ./deploy.sh
```

## วิธีใช้

```bash
deltascope serve   -listen :8080 -archive DIR -data DIR [-catalog F] [-rules F]
deltascope user    add|del|list <name>
deltascope catalog export > catalog.json
deltascope rules   export > rules.json
deltascope compare -a-start 2026-07-09T14:00 -a-end 2026-07-09T15:00 \
                   -b-start 2026-07-10T14:00 -b-end 2026-07-10T15:00 \
                   [-format text|json] [-all] [-no-color]
```

## ตรรกะการตัดสิน

- ตัวนับถูกเฉลี่ย**เป็นอัตรา**ต่อช่วงเวลา (ตรรกะเดียวกับ pmdiff)
- Δ% = (B − A) / |A| × 100; ตัดสินเมื่อ `|Δ| ≥ เกณฑ์` เท่านั้น (ค่าเริ่มต้น 15% เมตริกที่แกว่งมีค่าเฉพาะ)
- ขั้ว: `worse_up` / `better_up` / `neutral`
- A=0 → B≠0 รายงานเป็น ∞; เมตริกที่ไม่มีทั้งสองฝั่งจะถูกข้ามเงียบ ๆ
- เกิดใหม่ ⊕ / หายไป ⊖ เป็นเหตุการณ์ชั้นหนึ่ง แยกจากการเปลี่ยนขนาด

## ความปลอดภัย

PBKDF2-HMAC-SHA256 (600k รอบ) · เซสชันไร้สถานะลงนาม HMAC · จำกัดการล็อกอินราย IP · ไวต์ลิสต์ชื่อเมตริก + exec แบบอาร์เรย์ (ไม่ผ่าน shell) · CSP เข้มงวดไม่มี inline · systemd unit เสริมความแข็ง · ข้อมูลรับรองไม่ออกจากเครื่อง
